    • SDOC-JAZZ-INTEGRATION
    1. Jazz integration SDOC-JAZZ-INTEGRATION

    The StrictDoc OSLC RM server shall interoperate with Jazz as an OSLC RM 2.0 provider.
    • SDOC-JAZZ-ROOT-SERVICES
      2. Root services discovery SDOC-JAZZ-ROOT-SERVICES

      The server shall publish an RDF/XML OSLC root-services document that advertises the RM service-provider catalog and Jazz OSLC catalog descriptor.
    • SDOC-JAZZ-SERVICE-PROVIDER
      3. Service provider catalog SDOC-JAZZ-SERVICE-PROVIDER

      The server shall expose an RM service-provider catalog that lists the StrictDoc document service providers and their RM domain.
    • SDOC-JAZZ-INSTANCE-SHAPE
      4. Requirement instance shape SDOC-JAZZ-INSTANCE-SHAPE

      Each requirement representation shall advertise a provider-owned OSLC RM instance shape that describes its writable link properties.
    • SDOC-JAZZ-QUERY-SHAPE
      5. Query capability resource shape SDOC-JAZZ-QUERY-SHAPE

      The RM query capability shall advertise the same requirement resource shape that individual requirement representations use.
    • SDOC-JAZZ-OAUTH
      6. OAuth 1.0a discovery SDOC-JAZZ-OAUTH

      Root services shall advertise OAuth 1.0a consumer registration, request-token, authorization, and access-token endpoints compatible with Jazz.
    • SDOC-JAZZ-PUBLIC-URI
      7. Public resource identity SDOC-JAZZ-PUBLIC-URI

      Discovery, catalog, service-provider, dialog, shape, OAuth, and requirement links shall use the externally visible HTTPS origin.
    • SDOC-JAZZ-RDF
      8. RDF representations SDOC-JAZZ-RDF

      The server shall serve OSLC resource representations through RDF content negotiation and include OSLC-Core-Version 2.0.
    • SDOC-JAZZ-LINK-UPDATES
      9. Writable link updates SDOC-JAZZ-LINK-UPDATES

      The provider shall accept Jazz updates to provider-owned requirement links and return a response that Jazz recognizes as a successful save.
    • SDOC-JAZZ-RM-RELATIONS
      10. OSLC RM relation mapping SDOC-JAZZ-RM-RELATIONS

      The server shall map StrictDoc Decomposes, Elaborates, Specifies, Constrains, and Satisfies roles to their corresponding OSLC RM properties.
      • SDOC-JAZZ-STABLE-LINKS
        12. Stable requirement links SDOC-JAZZ-STABLE-LINKS

        Requirement relation targets shall use stable public requirement URIs so Jazz can navigate and retain links.
    • SDOC-JAZZ-INVERSE-RELATIONS
      11. Inverse relation projection SDOC-JAZZ-INVERSE-RELATIONS

      The server shall add the OSLC RM inverse relation to target requirements when reading a StrictDoc JSON export.
