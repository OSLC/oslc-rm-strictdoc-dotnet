    SDOC-JAZZ-ROOT-SERVICES
    2. Root services discovery
    UID: SDOC-JAZZ-ROOT-SERVICES
    RELATIONS (Parent):
    • SDOC-JAZZ-INTEGRATION Jazz integration (Decomposes)
    STATEMENT:

    The server shall publish an RDF/XML OSLC root-services document that advertises the RM service-provider catalog and Jazz OSLC catalog descriptor.
    SDOC-JAZZ-SERVICE-PROVIDER
    3. Service provider catalog
    UID: SDOC-JAZZ-SERVICE-PROVIDER
    RELATIONS (Parent):
    • SDOC-JAZZ-INTEGRATION Jazz integration (Decomposed by)
    STATEMENT:

    The server shall expose an RM service-provider catalog that lists the StrictDoc document service providers and their RM domain.
    SDOC-JAZZ-INSTANCE-SHAPE
    4. Requirement instance shape
    UID: SDOC-JAZZ-INSTANCE-SHAPE
    RELATIONS (Parent):
    • SDOC-JAZZ-INTEGRATION Jazz integration (Elaborates)
    STATEMENT:

    Each requirement representation shall advertise a provider-owned OSLC RM instance shape that describes its writable link properties.
    SDOC-JAZZ-QUERY-SHAPE
    5. Query capability resource shape
    UID: SDOC-JAZZ-QUERY-SHAPE
    RELATIONS (Parent):
    • SDOC-JAZZ-INTEGRATION Jazz integration (Elaborated by)
    STATEMENT:

    The RM query capability shall advertise the same requirement resource shape that individual requirement representations use.
    SDOC-JAZZ-OAUTH
    6. OAuth 1.0a discovery
    UID: SDOC-JAZZ-OAUTH
    RELATIONS (Parent):
    • SDOC-JAZZ-INTEGRATION Jazz integration (Specifies)
    STATEMENT:

    SDOC-JAZZ-PUBLIC-URI

    UID: SDOC-JAZZ-PUBLIC-URI
    RELATIONS (Parent):
    • SDOC-JAZZ-INTEGRATION Jazz integration (Specified by)
    STATEMENT:

    SDOC-JAZZ-RDF
    8. RDF representations
    UID: SDOC-JAZZ-RDF
    RELATIONS (Parent):
    • SDOC-JAZZ-INTEGRATION Jazz integration (Constrains)
    STATEMENT:

    The server shall serve OSLC resource representations through RDF content negotiation and include OSLC-Core-Version 2.0.
    SDOC-JAZZ-LINK-UPDATES
    9. Writable link updates
    UID: SDOC-JAZZ-LINK-UPDATES
    RELATIONS (Parent):
    • SDOC-JAZZ-INTEGRATION Jazz integration (Constrained by)
    STATEMENT:

    The provider shall accept Jazz updates to provider-owned requirement links and return a response that Jazz recognizes as a successful save.
